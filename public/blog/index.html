Our Application Stack

Posted on October 16, 2013

I'm going to introduce all the 3rd party products, frameworks and programming languages that are in use at BiblioFair. My main aims were simplicity, speed of development, and use of open source tools. I do not consider myself a very skilled programmer yet, but I really would like to contribute back to some of the awesome open source frameworks that make my job easier every day. As I didn't have any prior experience except with static websites, I was in good position to reach for the newest high-end tools.

The backbone of the site is the MEAN stack, which stands for MongoDb, ExpressJS, AngularJS, and NodeJS. As you might see there is a lot of JavaScript involved in there.

  • NodeJS is Chrome's JavaScript back-end runtime, which allows me to start the server with just about ten lines of code. It runs fast, is easy to use, and has a great number of packages that extend its functionality. I think it is great for developing web apps because you can share JavaScript front-end and back-end code. All checks that you make in the browser usually have to be done on the server too, so simple copy-pasting is just unbeatable here.
  • ExpressJS is one of the packages for Node. It's a package which makes it very simple to run a server and to route various requests. Listening on a server can be as simple as writing:
    var app = express(); app.listen(port, ipaddr);.
  • MongoDb is a NoSQL database. I can't compare it to a SQL database because I have never used one. In fact I'm taking a SQL database course at school right now. For this one I will refer you to the MongoDB website. To access MongoDB from Node, I use the MongooseJS package. Even though I has given me a hard time, I still think it is a great tool which is evolving very quickly and soon will become a serious competitor to the traditional database businesses.
  • AngularJS is a front-end JavaScript framework which among other things extends HTML functionality. It is an open sourced project by Google. There are still things that I have trouble to wrapping my head around. I don't know if it's just me or the bad documentation. But I really like the concept and even though it sometimes takes me a little longer to implement some functionality, I still think it has saved me time overall. Plus, it has been getting better as I understand more of it now.

That's for the main stack. All our design is built upon Bootstrap, a responsive UI front-end framework developed and open sourced by Twitter. Our server runs on the OpenShift free plan hosting.

I hope I haven't forgetten about anything important. If I have, leave a comment and I will expand the article accordingly.
